# Round 2 Challenge Answers & Steps

Here are the solutions for the 10 Round 2 advanced CTF challenges along with instructions on how to find each flag.

### Challenge 1: Base64 Decoded
**Flag**: `flag{b4s64_d3c0d3d}`
**How to find**:
1. Open the Developer Tools (F12) and inspect the page elements.
2. In the `<body>`, look for a hidden `<svg>` tag near the top: `<svg data-cipher="ZmxhZ3tiNHM2NF9kM2MwZDNkfQ=="...>`
3. Copy the base64 string `ZmxhZ3tiNHM2NF9kM2MwZDNkfQ==` and decode it (e.g., using a base64 decoder or the browser console `atob()`) to get the flag.

### Challenge 2: ROT13 Hidden
**Flag**: `flag{r0t13_h1dd3n}`
**How to find**:
1. Inspect the source code or use Developer Tools.
2. Search for a hidden `<p>` tag or script containing a ROT13 encoded string like `synt{e0g13_u1qq3a}`.
3. Decode the string using ROT13 (shifting letters by 13 places) to reveal the flag.

### Challenge 3: XOR Master
**Flag**: `flag{x0r_m4st3r}`
**How to find**:
1. Inspect the page source and look for an embedded `<script>`.
2. The script contains XOR logic with a specific key and an array of integers (e.g., `[...].map(x => String.fromCharCode(x ^ key))`).
3. Either replicate the script in your console or analyze the XOR logic to compute the decrypted string.

### Challenge 4: Caesar Shift
**Flag**: `flag{c43s4r_sh1ft}`
**How to find**:
1. Similar to the ROT13 challenge, look for a hidden element or embedded JavaScript with an obfuscated string.
2. It will use a Caesar cipher (a shift cipher other than 13).
3. Try different shift values (Brute-force all 25 possible shifts) on the encoded string until the recognizable `flag{...}` format appears.

### Challenge 5: Cookie Secret
**Flag**: `flag{c00k13_s3cr3t}`
**How to find**:
1. Open the Developer Tools and navigate to the **Application** tab (or **Storage** tab in Firefox).
2. Look under **Cookies** for the current domain.
3. You will find a cookie that was set by the challenge page containing the flag directly or base64 encoded.

### Challenge 6: Local Storage Flag
**Flag**: `flag{l0c4l_strg_fl4g}`
**How to find**:
1. Open the Developer Tools and go to the **Application** tab.
2. Look under **Local Storage**.
3. You will find a key-value pair generated by the challenge. The value is the flag.

### Challenge 7: Fake API Call
**Flag**: `flag{f4k3_4p1_c4ll}`
**How to find**:
1. Open the **Network** tab in Developer Tools and refresh the page.
2. The page makes a fake background API request (e.g., `fetch('/some-hidden-api')`).
3. Click the request and inspect the **Headers** (e.g., `X-Flag-Header`) or the **Response** payload to find the flag.

### Challenge 8: CSS is Awesome
**Flag**: `flag{css_1s_4w3s0m3}`
**How to find**:
1. Inspect the styles of the page. You can look at the embedded `<style>` block or the elements themselves.
2. Look for elements placed off-screen (`left: -9999px`) or styled with `display: none;` or check pseudo-elements like `::after` that have `content: 'flag{css_1s_4w3s0m3}';`.

### Challenge 9: URL Manipulation
**Flag**: `flag{u1r_m4n1p_fl4g}`
**How to find**:
1. Look at the URL structure or a hidden input form on the page.
2. You may need to add a parameter to the URL like `?debug=true` or alter the fragment `#admin` to reveal a hidden section containing the flag on the page.

### Challenge 10: Mixed Encoding
**Flag**: `flag{m1x3d_3nc0d1ng}`
**How to find**:
1. Inspect the source code and find the long encoded string.
2. This string requires multiple decoding steps. For example, it might be Base64 encoded first, and the resulting text might be ROT13 encoded.
3. Decode step-by-step: first decode Base64, then pass it through a ROT13 decoder to get the final flag.
